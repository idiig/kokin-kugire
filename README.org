#+title: Kugire Annotations for the Kokinwakashu

* Overview

A kugire (句切れ) annotation layer for the Kokinwakashu (古今和歌集) TEI/XML data.
Kugire marks a phrase or clause break within a poem. Two types are distinguished:

1. *Rule-based (grammar-wise)* (=morph=) — derived from morphological analysis;
   certainty gradient: =high= (終止・命令・已然・終助詞), =mid= (係助詞), =low= (体言止め等).
2. *LLM-assisted translation-based* (e.g. =kaneko=) — derived from an expert's
   explanatory translation via local LLM (Ollama =qwen2.5=), using a 3-step chain-of-thought:
   segment alignment → translation break detection → position mapping.
   The model returns ={"alignment": "...", "breaks": "...", "positions": [...]}=;
   the reasoning is recorded as comment lines in the cache file.

Multiple competing annotations are allowed on the same poem (different translators
may disagree). Each =<k>= element is tagged with its evidence source.

* Workflow

Two-phase pipeline:

1. *Batch tagging* (=kugire-tag=) — generate suggestions for all 1111 poems and
   write cache files + XML annotations automatically.
2. *Interactive review* (=kugire-review=) — open a specific poem's draft in nano,
   edit, and write back to XML.

Cache files (=tools/kugire/cache/kugire-{id}-{source}.txt=) are the persistent
draft record and are tracked in git. Commands must be run from =tools/kugire/=
so that the relative =cache/= path resolves correctly.

* Setup

#+begin_src sh
nix develop          # enters devShell; starts ollama serve automatically
ollama pull qwen2.5  # first time only (~4.7 GB)
#+end_src

* Usage

Build commands once from =tools/kugire/=:

#+begin_src sh
go build ./cmd/kugire-tag/
go build ./cmd/kugire-review/
#+end_src

Batch-tag all poems (run only one instance at a time):

#+begin_src sh
./kugire-tag --source morph   --xml ../../data/kokinwakashu.xml \
  --morph ../../data/morphological-annotation.txt \
  --kaneko ../../data/translation-kaneko.txt \
  --output ../../data/kokin-kugire.xml

./kugire-tag --source kaneko  # same flags; calls Ollama per poem
#+end_src

Review a single poem:

#+begin_src sh
./kugire-review --source morph 1   # opens nano in tmux window kugire-1
#+end_src

* Draft format

Each line of the poem may carry an end-of-line tag =​[K:<source>:<cert>]= marking a
kugire after that segment. Edit tags in nano to accept or adjust suggestions.
The =cert= field (=high= / =mid= / =low=) is set automatically for =morph= suggestions.
For =kaneko= drafts, the LLM's chain-of-thought reasoning appears as =# alignment:=
and =# breaks:= comment lines directly below the translation.

#+begin_example
年の内に
春はきにけり [K:morph:high]
ひとゝせを [K:morph:low]
こそとやいはむ [K:morph:high]
ことしとやいはむ
#+end_example

* Output schema

#+begin_src xml
<l n="1" xml:id="n1">
  <seg>年の内に</seg>
  <seg>春はきにけり</seg>
  ...
  <k n="2" source="morph" cert="high"/>
  <k n="3" source="morph" cert="low"/>
  <k n="4" source="kaneko"/>
</l>
#+end_src

- =@n= — 1-based segment number after which the kugire falls
- =@source= — evidence code (=morph=, =kaneko=, etc.)
- =@cert= — certainty: =high= / =mid= / =low=; omitted when unspecified (e.g. semantic sources)
