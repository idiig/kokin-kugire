#+title: Kugire Data Curation for the Kokinwakashu TEI/XML Data

* Introduction

#+begin_comment
TL;DR: Kugire (phrase breaks in waka) lack a public annotated dataset; each TEI project annotates from scratch. This project adds kugire as a separate annotation layer on the Kokinwakashu TEI/XML data. Separating the layer preserves annotation criteria and competing interpretations. The data enables structural and enjambment analysis and demonstrates layered, ambiguity-aware annotation practice.
#+end_comment

Kugire — phrase breaks within waka poems — are a basic structural element of waka, yet no standardized, publicly available kugire dataset exists [kamihiro]. For TEI/XML encoding of waka, each project must annotate from scratch.

This project adds kugire as an interpretive annotation layer to the TEI/XML data of the Kokinwakashu, one of the most important waka anthologies. Keeping kugire as a separate layer — rather than embedding it in the text — preserves the annotation criteria and any competing interpretations. The kugire data supports analysis of waka structure and enjambment, and serves as a case study in ambiguity-aware, layered interpretive annotation for literary and linguistic research.

* Related works

#+begin_comment
TL;DR: Existing kugire resources are scarce and single-source. [atuko315] provides a rule-based program and a Hyakuninisshu dataset, later applied to the Kokinwakashu, but cannot capture interpretive ambiguity. [okaasa] proposes two criteria — semantic independence (necessary) and grammatical break (sufficient) — both open to interpretation. Multi-source annotation with documented criteria is therefore needed. Applications include diachronic change of kugire position, poetic style, enjambment, and manuscript punctuation studies.
#+end_comment

Prior work on kugire is sparse. The few available resources treat annotation as a fixed, single-source task, yet the criteria proposed for kugire determination remain inherently ambiguous. Richer, multi-source annotation data would enable waka analysis across multiple dimensions: diachronic study of kugire, its relation to poetic style, enjambment, and the reading history of waka. The following sections review each area in detail.

** Kugire-related data

[atuko315] is one of the few existing kugire resources: it provides a program to identify kugire and a dataset for the Hyakuninisshu, a classical selection of one hundred waka poems by one hundred poets. The program was later applied to the Kokinwakashu and Shinkokinwakashu and compared against [okaasa] and [kamihiro]. The dataset, however, cannot capture the interpretive nature of kugire, which admits no single correct answer. A kugire dataset should therefore be a collection of annotations from multiple sources, each with documented criteria and interpretations.

** Kugire standard

[okaasa] proposes two criteria for kugire:

1. The necessary condition :: two or more semantically independent core sentences in the poem;
2. The sufficient condition :: a clear grammatical break in the poem.

Both conditions, however, remain open to interpretation. We therefore annotate kugire using rule-based and translation-based methods across multiple sources, implemented as a separate layer on top of the TEI data. This preserves information about whose judgment it is, what criteria were applied, and whether alternative interpretations exist.

* Data sources

#+begin_comment
TL;DR: The base data is [chin2026], a lexically extended TEI encoding of the Karoku manuscript built on [ikuura] and the Hachidaishu Vocabulary Dataset [Hodoscek2022]. Reference data for the rule-based method is the Hachidaishu Part-of-Speech Dataset [Yamamoto]; for the translation-based method, [kaneko]'s explanatory translation digitized as [yamamoto].
#+end_comment

** Base data

The base data is a lexically extended version [chin2026] of the TEI data of the Karoku second-year manuscript of the Kokinwakashu [ikuura]. [ikuura] describes the physical format of the manuscript and encodes various annotations — marginalia, apparatus, notes — and a list of authors with their social attributes; it also provides the segmentation of the waka text. [chin2026] maps tokenization from the Hachidaishu Vocabulary Dataset [Hodoscek2022] onto this segmented text, forming the basis for the kugire annotation.

** Reference data

For the rule-based method, we use the Hachidaishu Part-of-Speech Dataset [Yamamoto], which provides conjugation and part-of-speech data for each token. For the translation-based method, we use the explanatory translation of the Kokinwakashu by [kaneko], the earliest twentieth-century explanatory translation of the anthology, digitized and published on Zenodo [yamamoto].

* Output Data

#+begin_comment
TL;DR: The output is =kokin-kugire.xml=, the base data with =<k>= elements appended inside each =<l>=. Each =<k>= carries =@n= (1-based segment index), =@source= (=morph= or translator code), and optionally =@cert= (=high=/=mid=/=low=, rule-based only). Multiple =<k>= at the same position preserve competing interpretations without deduplication. The tag is project-internal; the target TEI encoding is not yet decided.
#+end_comment

The output is =kokin-kugire.xml=, a TEI/XML file derived from the base data by appending kugire annotations to each poem. The original text, segmentation, and all existing TEI markup are preserved without modification; the annotation layer is additive.

Each kugire position is encoded as a =<k>= element placed after the final =<seg>= child of the corresponding line element =<l>=:

#+begin_example
<l n="1" xml:id="n1">
  <seg>年の内に</seg>
  <seg>春はきにけり</seg>
  <seg>ひとゝせを</seg>
  <seg>こそとやいはむ</seg>
  <seg>ことしとやいはむ</seg>
  <k n="2" source="morph" cert="high"/>
  <k n="3" source="morph" cert="low"/>
  <k n="4" source="morph" cert="high"/>
  <k n="2" source="kaneko"/>
</l>
#+end_example

The rule-based method identifies three candidate positions. The strongest (=cert="high"=) is after the second segment, where /けり/ appears in the conclusive form, ending the first complete sentence. A weaker candidate (=cert="low"=) is also placed after the third segment. The LLM-assisted translation-based method, drawing on [kaneko]'s explanatory translation, identifies only the second segment as a kugire. The competing candidates are preserved independently.

The =@n= attribute gives the 1-based index of the segment after which the break falls; =@source= identifies the annotation source (=morph= for the rule-based method, =kaneko= for the LLM-assisted translation-based method). Rule-based annotations carry a =@cert= attribute (=high=, =mid=, or =low=) reflecting the strength of the morphological evidence; translation-based annotations do not, because the translation-based criterion does not yield an analogous confidence gradient.

Multiple =<k>= elements at the same position represent competing interpretations — from the same source at different grammatical strengths, or from different sources — and are preserved as-is. No normalization or deduplication is applied.

The =<k>= element is a project-internal tag pending a decision on the target TEI encoding.

* Annotation pipeline

#+begin_comment
TL;DR: The annotation follows a four-step pipeline: rule-based suggestion from morphological data, LLM-assisted translation-based suggestion from [kaneko]'s translation, human review in nano, and rendering back to the TEI data (Figure 1). The rule-based method classifies candidates by strength: strong candidates end in the conclusive form, moderate candidates exhibit a copula or information-structure shift, and weak candidates carry a sentence-final particle with remaining semantic interdependence across segments (Figure 2). The LLM-assisted translation-based method uses =qwen2.5= served locally via Ollama, with a prompt that instructs a three-step chain-of-thought: segment-to-translation alignment, break detection in the translation, and mapping of break points to segment positions in the original poem. The prompt includes three few-shot examples covering no-kugire, second-segment, and fourth-segment cases. The model returns a JSON object with =alignment=, =breaks=, and =positions= fields; the reasoning is recorded in the cache file alongside the suggested positions for the human reviewer.
#+end_comment

The annotation follows an LLM-assisted pipeline:

1. Rule-based suggestion of candidate kugire positions based on [Yamamoto]
2. LLM-assisted translation-based suggestion of candidate kugire positions based on [kaneko, Yamamoto]
3. Review and correction in a pop-up editor showing the original poem and its reference data
4. Rendering the confirmed annotation into the TEI data

The pipeline accelerates the annotation process. The per-poem review cycle reduces cognitive load and improves accuracy [reference].

** Rule-based (grammar-wise) kugire suggestion

[atuko315] uses conjugation information to identify candidate kugire positions based on three criteria:

1. Phrases ending in the conclusive, imperative, or realis form (the first and third suffix groups in gakkō bunpō)
2. Phrases ending with the copula, e.g., =nari=, =tari=
3. Phrases ending with sentence-final particles expressing question, prohibition, or exclamation

Our suggestion extends this approach by allowing multiple candidates and classifying them into strong, moderate, and weak.

Strong candidates are phrases ending in the conclusive form. For example, in the first poem of the Kokinwakashu, the past-tense auxiliary verb /keri/ appears in the conclusive form, indicating a grammatical break after it.

#+begin_example
年の内に / 春はきにけり / 一とせを / こぞとやいはん / ことしとやいはん

tosi=no uchi=ni / paru=pa k-in-iker-i / pito+tose=wo / kozo=to=ya ip-am-u / ko+tosi=to=ya ip-am-u

year=gen inside=dat / spring=top come-mnf-hps-ccl / one+year=acc / last.year=cmp=xcl say-cjt-ccl
/ this+year=cmp=xcl say-cjt-ccl

translation: Within the year / spring has come! / one year / shall (we) call (it) last year? / shall (we) call (it) this year? 
#+end_example

Moderate candidates are phrases where the copula or information structure suggests a break, though not conclusively. In the second poem of the Kokinwakashu, the segments before and after =wo= are syntactically connected, yet the former presents a scene while the latter offers a conjectural statement about it — a shift that may indicate a break.

#+begin_example
袖ひちて / むすびし水の / こほれるを / 春立つけふの / 風やとくらん

sode pit-ite / musub-isi mizu=no / kopor-er-u=wo / paru tat-u kepu=no / kaze=ya tok-uram-u

sleeve be.wet-seq / scoop.up-pst.adn water=gen / freeze-sta-adn.nl=acc / spring start-adn today=gen / wind=xcl melt-cjt-ccl

translation: making sleeve wet, / water (I) scooped up / has frozen, while / spring starts, starting today's / wind! would (you) melt (it)? 
#+end_example

Weak candidates are phrases where a sentence-final particle suggests a break, but the surrounding segments remain semantically interdependent. In the third poem of the Kokinwakashu, the question particle in the second segment may suggest a break, yet the following segment answers that question — semantic interdependence that makes the break uncertain.

#+begin_example
春霞 / たてるやいづこ / みよしのの / 吉野の山に / 雪はふりつつ

paru+kasumi / tat-er-u=ya izuko / mi-Yosino=no / Yosino=no yama=ni / yuki=pa pur-itutu

spring+haze / rise-sta-ccl=xcl where / hpf-Yosino=gen / Yosino=gen mountain=dat / snow=top fall-itr

translation: Spring haze / where is (it) rising /  Yoshino / on the Mountain Yoshino / snow is falling
#+end_example

Each judgment is cached per poem for the review step and may in the future be encoded as a separate TEI element. The rule-based suggester takes the original poem and its morphological and lexical data [Yamamoto] as input, outputting candidate kugire positions.

** LLM-assisted translation-based kugire suggestion

We also use a large language model (LLM) to suggest candidate kugire positions from the explanatory translation of [kaneko] as encoded by [yamamoto]. Where the rule-based approach draws on morphological and grammatical features, the LLM-assisted translation-based approach draws on the contemporary Japanese translation, capturing breaks rooted in the translator's reading rather than grammatical form alone.

The LLM is =qwen2.5=, served locally via Ollama [ollama]. The model receives the five segments of the original poem and the corresponding explanatory translation, and outputs a JSON object with three fields: =alignment= (step 1), =breaks= (step 2), and =positions= (the resulting candidate indices).

The prompt instructs the model to follow a three-step reasoning process:

1. *Segment alignment*: Identify which segments of the original poem correspond to which parts of the explanatory translation.
2. *Break detection*: Locate points in the translation where the translator ends a sentence.
3. *Position mapping*: Map the identified break points to the corresponding segment boundaries in the original poem.

This mirrors the workflow a human annotator follows when using an explanatory translation as a reference.

The prompt includes three few-shot examples from real Kokinwakashu poems, each demonstrating the same three-step reasoning chain: one with no kugire, one with a second-segment break, and one with a fourth-segment break.

The model's JSON output is parsed, and valid segment indices are converted to kugire positions; invalid or out-of-range indices are discarded. Each position is recorded in the TEI data as, e.g., =<k n="2" source="kaneko"/>= , independent of positions from the rule-based method.
